- Plan migration projects by gathering information about on-premises data centers
- Server utilization data and dependency mapping are important for migrations
- [[Agentless Discovery]]
	- VM inventory, configuration, and performance history for CPU, memory, and disk usage
- [[Agent-based Discovery]]
	- System configuration, system performance, running processes, and details of the network connections between systems

- Results can be viewed with the AWS [[Migration Hub]]
- Simplest way to move is using the AWS [[MGN]] or Application Migration Service
- Utilizes lift-and-shift (rehost) solution which simplifies migrating applications to AWS, highly compatible, minimal downtime, reduced costs
